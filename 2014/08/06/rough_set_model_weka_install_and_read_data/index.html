摘要

本文是Rough set model粗糙集理论系列第一篇文章，主要介绍如何在Eclipse中安装weka资源包，以及读取arff数据

Rough Set model简介

有关Rough Set的英文介绍，请移步Wikipedia。这里说的比较详细，但有的地方可能不够准确。比如例子中，O1和O2是重复的，因此在后面解释的时候，只能用information table而不用集合的概念（集合中的元素必须是互异的。以后的研究中涉及到求集合的度（Card），Wiki中的这个例子可能造成一定的误解）。

中文一般将Rough Set翻译为粗糙集（百度百科）或粗集合（维基百科）。

除了Rough Set之外，Rough Set Model或Rough Set Theory也经常用到。在搜索相关文献时尽量采用Rough Set作为关键词。

weka简介

Weka是新西兰Waikato大学开发的一款用于数据挖掘（Data Mining）的开源软件包。安装文件与开发包均可在其网站下载。目前支持Windows，Mac与Linux系统，基于Java开发。由于版本众多，请各位自行查看下载。

Weka的功能不如Matlab强大，但是优点在于其完全开源，并且方便进行拓展开发。博主选用Weka另一个原因是厌倦了C++，尤其是C++对开发环境的挑剔（我这种小白经常换一台电脑，C++代码就挂掉了，然后要用一天的时间重新配置环境）。

在这里再贴出另一个非常常用的Rough Set专业开源程序Rosetta，基于C++开发，目前许多有关Rough Set的研究学者都在使用。

对于只想要应用weka的情况来说，能够使用Weka Explorer就足够了，这是一个界面化的数据挖掘工具，提供了非常多的常用功能以及机器学习模型。并且有相当多的中文教程可供参考。

对于需要使用Weka提供的相关接口进行深入开发的情况，需要将Weka资源包导入Eclipse（大神当然也可以不使用集成开发环境），调用相关接口进行开发。Weka提供了相当多的接口，但是目前除了网站上提供的一个英文文档之外，相关学习资料不多。

如果您想学习使用Weka，并且是一个代码和数据挖掘双料小白，博主推荐一本英文书：Data Mining: Practical Machine Learning Tools and Techniques，博主看的是第二版，最新的第三版会多一些内容。

当然还是那句话，如果你是大神，请提出宝贵意见！

读取arff数据

所谓数据挖掘，当然是基于数据来说的，没有数据一切都是空谈。

实验中遇到的数据分为两种：人造数据与现实采集数据。人造数据可以是随机产生，也可以是为了验证某些功能而编造出来。现实采集数据如果细分，可以有原始数据和生成数据两种，但本质上它们都是从现实中得到的数据。

实验的第一步通常都是加载数据，这里，通过编写实现加载数据的功能，来验证Weka安装是否成功，也是以后代码的基础。（本人代码写的很烂，基本没有优化，如有好的建议，请通过邮件或者QQ联系本人，感激不尽！）

  • 什么是arff
    arff是一种文件格式，并且该类型文件可以由MS Word打开。这种格式是Weka推荐的数据存储格式（吐槽：我真没觉得有什么优点……一点都不智能的说）。
    arff文件中包含了所有数据信息，包括数据名称，数据项（Attribute或者Feature，本文以后都用Feature代替）名称，以及所有的数据（本文以后用Instance代替）。

  • 编写数据文件
    一般来说，数据首先需要存在一个特定文件中，才能被正确读取。Weka数据输入文件格式的复杂是经常被人吐槽的。这里，介绍比较简单的一种数据文件制作方法，该方法也是Data Mining书中介绍的方法。

    1. 首先，新建一个Excel文档。将数据以及数据信息放入其中。文件名为数据名称，第一行为Feature名称，从第二行开始输入数据（Instance），然后将文件保存为CSV格式（保存时可能提示该格式可能丢失一些符号，不用理他就行）。示例见下图。该数据会在以后的代码中用到。该数据是来源于参考文献 1 。
    2. 打开Weka Explorer，点击Open File，先将类型选到csv，然后找到刚保存好的csv文件，点击open。
    3. 如果一切顺利，应该能看到刚编写的数据文件已经加载到了Weka中了。其实按照Weka自己说的，csv文件也是支持使用的。但是为了以后能够更方便的使用数据，建议将csv文件转换为arff文件。非常简单，只要点击右上角的save就可以了。保存的arff文件放入一个英文目录的文件夹，方便以后使用。我将文件放在D盘根目录下。

  • 编写代码读取数据
    首先，使用Java的IO读取arff文件

    File inputFile = new File("D:\\German.arff");

    然后，构建一个ArffLoader对象loader，并调用setFile方法，读取File对象

    ArffLoader loader = new ArffLoader();
loader.setFile(inputFile);

    调用ArffLoader的getDataSet()方法，将返回一个Instances类的对象
    在Weka中，数据是存放在Instances类的对象中的，可以将Instances理解为一张数据表。下面一节将详细讨论Instances类。

    Instances ins=loader.getDataSet();

    至此，读取工作完毕，以后的代码都是围绕Instances来展开。当然不要忘记import相关的包

    import weka.core.*;
import weka.core.converters.*;

    如果你想验证一下代码是否正确，可以将数据打印出来。Weka为Instances添加了print方法。

    System.out.print(ins);

    你将在控制台看到数据的构成，包括数据名称，Feature名称，以及各项数据。

Instances类

Instances类是Weka中非常重要的一个类，它存储了整个数据表。这里来看一下几个重要的方法：

  • 构造函数
    前面几个构造函数需要读取一个Java IO中的Reader，暂且放到一边；
    下面就会发现一个拷贝构造函数

    public Instances(/* @non_null@ */Instances dataset)

    这个函数其实就是复制一份数据给新的Instances。由于Java在赋值时使用的是类似C++中引用的方式，如果直接对Instances对象使用赋值操作，新的Instances是原有Instances的一个别名，两者指向相同的数据区（请参照其他Java教程，或参看书[2]P49关于赋值的解释）。因此，在以后的操作中，可以利用这个拷贝构造函数来解决这一问题。

    接下来的构造函数

    public Instances(/* @non_null@ */Instances dataset, int capacity)

    主要功能是从原有的Instances构造一个新的Instances，区别在于新的数据表为空，只复制Attribute信息

    public Instances(/* @non_null@ */Instances source, int first, int toCopy)

    这个构造函数从原有的Instances中复制一部分数据

  • Attribute有关方法
    在Instances中，每一个Feature用Attribute表示，用一个FastVector（和vector差不多，只不过是Weka自己设计的，博主也没再深入的看）来存储。而这个FastVector的属性是protected，因此要想在外部得到Attribute就必须调用其他public的方法。

    public/* @pure@ */Attribute attribute(int index)

    这个方法有够简洁明了。参数表示Attribute的位置（从0开始），返回一个Attribute类型。

    public void insertAttributeAt(/* @non_null@ */Attribute att, int position)

    在指定位置添加一个Attribute，并且给里面所有值置空（null）。

    public void deleteAttributeAt(int position)

    很重要的方法。删除指定位置的Attribute。这里要多说几句。当想要得到一个新的Instances的时候，我倾向通过删除Attribute来实现。而另一种方法，即新建一个空的Instances然后添加Attribute会比较麻烦，因为初始的Instances在指定容量（capacity）以后是完全空的，而刚才提到的insertAttribute方法是不能给数据赋值的，这样就需要遍历所有的数据来进行赋值操作。而使用删除操作是直接将一个Attribute的指针置空，从Java执行效率上来说是快一些的。当然，博主只是一个菜鸟，希望能有高手来指正。

    public/* @pure@ */int numAttributes()

    返回该Instances中Attribute的数量。

  • Instance有关方法
    注意看仔细了！这个Instance没有s！这里的Instance是表示每一条数据。由于Attribute实际是不存储数据值的，数据值都存在Instance当中。

    public/* @non_null pure@ */Instance instance(int index)

    简介明了返回一个Instance。这里需要看一下Instance类中的一个方法

    public /*@pure@*/ double value(int attIndex)

    这个方法返回了Instance里某个Attribute的值。在以后的操作中，凡是要取某个特定位置的值，都要使用这个方法。
    另一个Instance的方法是

    public void setValue(int attIndex, double value)

    这个方法的作用是设置（或更改）某个值。注意这里用的是double，这个方法的一个重载是用的string，这意味着数据值只有这两种：double与string。这一点同样也可以通过Instance中value与stringvalue两个方法的返回值看出。因此在调用这些方法的时候一定要注意数据类型。

    public/* @pure@ */int numInstances()

    回到Instances类，这个方法是返回Instance（没有s）的数量。

Instances类中重要的方法暂时就介绍到这里，以后需要用到的函数会到时候再补充。
