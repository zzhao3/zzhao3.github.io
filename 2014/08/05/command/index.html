概要

本文列举一些常用命令，以及可能需要注意的地方，主要用来提醒自己作为备忘。各项解释均系自己编造，有不对的地方欢迎留言指正。

Hexo常用命令

所有命令必须在hexo目录下。（小白解释：即在你的电脑中，hexo文件夹里，右键->git bash然后弹出命令行窗口）

hexo g 编译生成（generate）似乎是先清理再生成

hexo s 本地发布（server） 可在本地通过浏览器查看，浏览器地址 http://localhost:4000/ 端口号可能根据你的配置不同而改变，默认为4000

hexo n nameofnewfile 添加一篇新博客（new）。其中nameofnewfile可以替换为其他文件名。注意这里输入的只是新博客所存储的文件名（相当于你word文档的名称），与你要发布博客的标题无关（当然为了方便你查找，最好相关）。

hexo d 部署（deploy）。即将本地修改好的个人主页“上传”到你的github中。

博客文档格式

博客编辑时使用Markdown语法。具体语法格式请参照其他链接。

编辑博客时请不要使用windows中记事本编辑器。推荐使用Notepad++或其他“高大上”编辑器。本人使用的是UltraEdit。可能需要在编辑时转换编码格式为utf8，否则在网页中会显示乱码。

Markdown语法

本段供作者自己备忘使用，将持续补充。具体语法详情参见 Markdown语法说明

斜体

在需要斜体的字句两头加上下划线”_”或者星号”*”。

列表

列表前使用星号“*”或者加号“+”来使改行变成列表。注意星号后面需要有一个空格。

代码区块

在代码区块前使用Tab制表符，使得该段变成代码段。或者使用四个空格，也能达到相同效果。
