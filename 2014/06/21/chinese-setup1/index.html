简介

本文主要参照cnFeat以及他的另一个网页cnFeat的相关工作。但在其博文中有些许的错误。同时，也参照了其文中引用的一些内容，这些一并在文中会予以注明。

本文只适用于初学者。所谓初学者是指对于个人建站毫无认识，并且对于代码编写毫无经验的人。文中将详细描述如何完成每一步，但不会深入介绍相关概念与功能，只是尽量标注出相关资料链接。

为响应建设和谐社会的号召，文中所提供的某些网站可能无法打开，由于作者无法亲自试验，请遇到问题时联系我。

由于大多数网站位于国外，因此至少需要一定的英文基础。

准备

本节介绍相关准备工作，包括必要软件安装，相关网站的注册。

Github注册

Github本意是为项目合作开发提供的一个开放平台。每个用户可以将自己的代码放到个人主页中，并允许其他人阅读以及补充，适用于远程的合作开发。而目前更为吸引人之处在于，许多大牛在上面发表自己的新观念与观点，对于计算机专业或对编程感兴趣的人很有帮助。缺点在于，由于计算机的范畴太广，而这个较为新的网站又没有足够的用户，大家主要集中于网站开发与算法研究，某些方向可能参与者较少。

在这里，我们用github免费提供的空间，来发布个人主页。因此，必须要注册一个用户。网址 https://github.com/ 或使用前面的链接（注意是https的安全链接）。在第一个方框中输入用户名（昵称，英文），第二个输入邮箱地址（我用的hotmail，国内邮箱地址没有测试，有可能收件速度较慢），第三个填写密码（至少有一个小写字母，一个数字，长度不低于7位）。然后点击下面的“sign up for Github”
之后可能会有邮件验证。

注册完成之后，登陆你的github。点击右上角的一个+号，会出现两个选项，选择“new repository”（有人翻译成新建仓库）。

填写项目信息：

project name：zzhao3.github.io

description： This is my space

注1：这里zzhao3要换成你自己github的用户名
注2：Github Pages的Repository名字是特定的，比如我Github账号是zzhao3，那么我Github Pages Repository名字就是zzhao3.github.io。

安装相关软件

必选软件

要使用hexo，则必须安装相关软件。hexo的官方主页上清楚的讲述了如何安装。正规安装方法请移步hexo安装

这里介绍了Windows和Mac两种操作系统下的安装方法。建议按照官方文档进行安装。

还要说明的是，Mac下必须先安装Xcode，否则在安装过程中会自动下载安装Xcode。该软件可以在app store中找到，比较大，可能会需要较长时间。

可选软件

Windows:

文本编辑器Notepad++。免费的文本编辑器，在后面会用到来打开一些文本文件，以及编写博客。

Mac:

文本编辑器Brackets免费的Mac下文本编辑器。

Markdown编辑器MacDown同样免费。

注：其实很多开源免费的东西都不错，何必选用盗版。

Github配置

ssh配置

ssh这个东西对于一些小白来说比较陌生，这里您就给他看做本地与网络服务器的一个连接验证即可。在以后做链接时，如果本地的ssh证书与网络上的证书匹配，那么则视为一次安全链接。否则，你本地的文件将无法部署到网络中。

首先，生成新的ssh key。打开命令行窗口（windows里“运行”窗口输入cmd，mac在application->utility->Terminal），然后输入

ssh-keygen -t rsa -C "邮件地址@youremail.com"

注1：这里的邮箱地址填写您自己注册github用的。

注2：windows下不好直接粘贴，要点cmd窗口的左上角，然后选编辑，然后里面有个粘贴。

Generating public/private rsa key pair.
Enter file in which to save the key 
(/Users/your_user_directory/.ssh/id_rsa):<回车就好>

然后要求你输入用户名密码之类的。第一次用命令行的小白注意，输入密码后什么也不会显示，你就闷头输入然后按下回车就好了，不要害怕。

最后，在命令行窗口里会出现一个方框套一些字母的图案（木有截屏）。恭喜你成功了。

添加ssh到github

下一步需要将你刚才生成的ssh key添加到github中。ssh存储在

  1. windows用户C:\Documents and Settings\Administrator\.ssh\id_rsa.pub (看不见的话需要显示隐藏文件)
  2. mac用户 /Users/你的用户名/.ssh/id_rsa.pub (.ssh是隐藏文件夹，如果要看需要输入ls -la)（可以用vi打开，vi id_rsa.pub 关闭的时候输入:q!）

复制里面的内容，注意不要多带空行。

登陆github系统。点击右上角的 Account Settings—->SSH Public keys —-> add another public keys。把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了

之后可以通过在命令行输入

ssh -T git@github.com

来测试是否配置成功。如果显示类似下面的字样

The authenticity of host 'github.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

没关系，输入yes按回车，如果提示

Hi zzhao3! You've successfully authenticated, but GitHub does not provide shell access.

恭喜你成功了.

设置用户信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。

Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己github的.

git config --global user.name "zzhao3"//用户名
git config --global user.email  "zzy198700@hotmail.com"//填写自己的邮箱

hexo配置

hexo安装

hexo是一款博客发布工具，并能够提供多种博客模板（这里称为主题theme）。主题可以通过在hexo官网下载，也可以在github中提取。这里我们采用github提取的方法。

首先打开命令行窗口。windows用户需要新建一个叫Hexo的文件夹，然后在该文件夹中右键->Git Bash。mac用户在新建Hexo文件夹后，需要在命令行窗口中移动到该文件夹。

首先，安装hexo

npm install -g hexo

安装成功之后，执行

hexo init

这样，hexo的基本配置已经完成了，你可以执行下列命令

hexo g
hexo s

然后打开浏览器（不推荐IE），在地址栏输入localhost:4000，就可以看到一个hexo自带的默认主题。

注：有的小白会发现，这时命令行窗口竟然不能输入了！没关系，按下ctl+c就能够终止该进程。

jacman主题

这里我选用的是WuChong的jacman主题。有关该主题的详细情况请参照其博客如何使用jacman主题。

在Hexo文件夹下执行

git clone https://github.com/wuchong/jacman.git themes/jacman

要想启用该主题，需要修改Hexo目录下的_config.yml配置文件中的theme属性为jacman。

以后想要更新主题的话，可以在jacman目录下执行

git pull origin master

别忘了先保存你原来的文件，尤其是jacman目录下的_config.yml

这时就可以查看jacman主题了。还是命令

hexo g
hexo s

有关更详细的如何配置jacman方法，请参见如何使用jacman主题
有关如何个性化主页，请参见hexo与jacman个性化配置

部署

所谓部署，即给你在本地建立的个人主页放到互联网上，并通过github提供的服务发布给全世界。推荐查看hexo官网提供的部署方法.

看不明白也没关系，这里为您介绍一种比较直白的方法。

首先安装hexo-deployer-git。在Hexo目录下执行

npm install hexo-deployer-heroku --save

然后修改hexo目录下的_config.yml

deploy:
 type: git
 repo: <repository url>
 branch: [branch]
  message: [message]

其中，repo后面填写你github中项目的地址，branch为master，message可以不写。我自己项目的示例：

deploy:
  type: git
  repository: https://github.com/zzhao3/zzhao3.github.io.git
  branch: master

给zzhao3替换成你的github用户名也成。

配置好之后，执行命令

hexo d

来进行提交部署。

注意：必须先在本地生成，并验证正确之后再提交。

有关hexo中的命令，请参见利用Github建立hexo个人主页常用命令

如果一切顺利，你可以在.github.io看到你发布的主页。（例如我的是zzhao3.github.io）

个人域名

所谓域名，可以理解为一个ip地址的名字。在刚刚完成的步骤中，只能通过带有github的特定域名来进行访问。如果想要自己来设置这个“网址”，那么需要申请并购买域名。

域名购买

由于域名由万恶的美帝控制，天朝人民想要自己来购买域名需要去美帝的网站。我个人使用的是godaddy

懂英文的同学可以自行购买。由于本人不在国内，没有办法测试如何在国内购买。推荐cnFeat的相关教程。（好像可以用支付宝的说）

github域名绑定

在github你的.io项目里面，新建一个文本文档，命名为CNAME。

在该文档中填写你申请的域名。比如我的文档中就填写zhaozongyuan.com
